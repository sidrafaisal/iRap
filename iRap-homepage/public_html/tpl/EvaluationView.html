Evaluation

To evaluate the proposed approach, we performed experiments on the iRap framework using changesets published by DBpedia and compared the results with the DBpedia Live Mirror tool. The comparison considers two cases: using iRap to update a previously-established local replica of

  • an entire remote dataset
  • a subset of a remote dataset

These two cases simulate two ways in which iRap can be used:

  • using interest-based changeset propagation for future updates of a local copy of a large dataset
  • starting with a new subset of the large dataset.

We set up two target datasets and potentially interesting dataset using Jena TDB and jena-fuseki for each dataset. The potentially interesting dataset stores potentially interesting triples for each interest expression within a named graph. All experiments were carried out on a 64-bit machine with Windows 7, Intel(R) Core i7-4770 CPU, 16GB RAM and 1TB HD.

We tested our approach using the DBpedia dump for the initial setup of the target datasets for two different application domains, namely, Location and Football datasets. For study purposes, the evaluation material can be found at iRap's GitHub repository.

An thorough evaluation of the approach, using large-scale real-world data dumps and changesets regularly provided by a renowned LOD dataset, indicates that our method can significantly cut down on both the size of the data updates required to consistently maintain a localized dataset replication up-to-date, as well as the speed by which such updates can take place.
